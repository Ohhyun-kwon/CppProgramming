## Q1. 접근지정자가 왜 필요한가?
A1. 캡슐화, 정보은닉 등을 구현하기 위해 public, private, protected 의 접근지정자가 필요하다.

## Q2. 생성자와 setter 멤버함수는 모두 멤버변수를 설정하는 기능이다. 차이점을 설명하라.
A2. 생성자는 객체가 생성될 때 한번만 호출되는 멤버함수이기에 초기값을 설정하기 위한 함수이다. setter 멤버함수의 경우 필요할 때마다 호출하여 값을 설정하는 등의 기능을 할 수 있다는 차이점이있다.

## Q3. 객체지향형 언어의 특징인 캡슐화와 정보은닉을 설명하라.
A3. 캡슐화는 변수와 함수를 하나의 클래스로 묶어서 외부로부터 보호하는것이고 정보은닉은 캡슐화의 일부로 데이터를 외부에서 직접 접근하지 못하게 접근 지정자를 통해 보호하는것을 뜻한다.
