Q1. 알고리즘 설계와 코딩 중 무엇이 중요하고 그 이유는 무엇인가?  
A1. 알고리즘 설계가 더 중요하다고 생각하고 그 이유로는 알고리즘의 경우 문제를 해결하기 위한 논리이기에 같은 답을 내더라도 효율이나 정확도에 차이를 보일 수 있지만 코딩의 경우 정해진 문법과 컴파일러등의 보조 장치들로 오류를 찾는데 보다 쉽다.

Q2. 클래스와 객체가 무엇인지 설명하시오.  
A2. 클래스는 데이터와 메서드를 하나의 단위로 묶어 정의한 사용자 정의 자료형이며, 객체를 생성하기 위한 추상적인 설계 구조이다. 반면 객체는 클래스에 정의된 구조를 기반으로 생성되어 실제 메모리 공간에 존재하고, 독립적인 상태와 데이터를 가지는 프로그램 구성 요소이다.

Q3. 절차지향 프로그래밍 언어와 객체지향 프로그래밍 언어의 차이를 조사하라.  
A3. 절차지향 프로그래밍은 프로그램을 순차적인 절차와 함수의 집합으로 구성하는 방식이다. 데이터와 함수가 서로 분리되어 존재하며 프로그램의 실행 흐름을 중심으로 구조가 구성된다. 데이터는 여러 함수에서 직접 접근하거나 수정할 수 있으며 프로그램의 규모가 커질수록 데이터의 상태를 관리하기 어렵고 유지보수가 복잡해질 수 있다. 객체지향 프로그래밍은 프로그램을 독립적인 객체들의 집합으로 구성하는 방식이다. 객체는 데이터와 이를 처리하는 메서드를 하나의 단위로 묶어 관리한다. 데이터는 객체 내부에 저장되며 정해진 메서드를 통해서만 접근할 수 있다. 또한 상속과 같은 개념을 통해 기존 구조를 확장하고 재사용할 수 있어 코드의 재사용성과 유지보수가 간편하다.

Q4. 라이브러리가 무엇인지 자세히 설명하시오.  
A4. 라이브러리는 프로그램 개발 과정에서 자주 사용되는 기능들을 미리 구현하여 모아 놓은 코드의 집합으로 프로그램의 구조를 효율적으로 구성하기 쉽다.

Q5. 라이브러리가 목적파일 형태로 존재하는 이유를 설명하라.  
A5. 라이브러리가 목적파일 형태로 존재하는 이유는 이미 기계어로 번역된 상태이므로 사용 시 재컴파일이 불필요하여 빌드 시간을 단축할 수 있으며, 기계어로 번역된 상태이므로 소스 코드를 확인하기 어려워 지식재산을 보호할 수 있다.

Q6. 링크 과정이 필요한 이유를 설명하라.  
A6. 큰 프로그램의 경우 여러 소스 파일로 구성되어 있으며 각 파일은 컴파일 과정을 거쳐 목적파일로 번역된다. 이때 생성된 목적파일들은 서로 독립적으로 존재하기 때문에 실행할 수 없다. 따라서 이러한 목적파일들과 필요한 라이브러리를 하나의 실행 파일로 결합하기 위해 링크 과정이 필요하다.
